【前文：思想背景（追加・確定）】
本プロダクトは、過去に検討された「The Forecast Check」を含む試行錯誤の中で、以下の問題意識に基づいて成立している。
世の中には、多くの「予測」が発せられているが、その多くは結果と照合されないまま消費・忘却されている。成功した予測や象徴的な的中例のみが記憶され、外れた予測・未検証の予測は記録に残らないという構造的偏りが存在する。
この偏りは、予測者の評価や断罪以前に、「事実としての履歴が存在しない」こと自体が問題である。
本プロダクトは、

予測を評価するためではなく、
人を格付けするためでもなく、
正しさを裁定するためでもない。
予測と結果を分離し、照合可能な形で記録し続けること——その行為そのものに価値がある、という立場を取る。
この思想は、**感情や権威ではなく、事実を見る態度（FACTFULNESS的態度）**と整合する。なお本プロダクトは、照合結果を **的中率** として可視化し、予測者別のランキング表示を行うことがある。ただし、断罪・誹謗・煽りを目的とした表現や設計は行わない。

0. このサイトは何か（1分で分かる説明）

このサイトは、格闘技（主にRIZIN/UFCなど）の試合・興行に関する予測と、その検証ログを蓄積し、後から検証できる形で公開することを目的とする。
重要なのは「当たった／外れた」だけではなく、いつ・どんな根拠で・どの形式で予測したか
結果が出た後に どう検証し、学びを更新したか
を、データとして残すこと。

サイトは GitHub Pages でホスティングしている。


1. 絶対に守る運用ルール（最重要）
1.1 ログの正本（唯一の真実）

セーブデータ v1.2.1（完全統合版・全文）が唯一の正本。
運用方針は 毎回「全文更新」。
差分更新・簡略化は禁止。
1.2 データ単位

原則：1試合 = 1 JSON（standard-log-v1.2-json）。
ただし、興行単位（例：RIZIN.20 など）でまとめる運用を採用する場合は、「1興行 = 1ファイル」 で統一し、中途半端な混在（試合単位と興行単位が混ざる）を避ける。
1.3 予測の扱い

主予測：勝者方向（誰が勝つか）
副次情報：KO / 判定 / SUB を補助情報として持たせる。
「何を予測として採用するか」「どこまでを副次情報とするか」は、この枠組みを崩さない。

1.3.1 予測者（誰が予測したか）の扱い（追加・確定）

予測は「誰が・いつ・何を」言ったかを一次情報として残す。表示上も、予測者（予測主体）の情報を予測データのメタ情報として扱い、イベント/試合単位で参照可能にする（未確定・未定義の場合は **「不明」** と明示する）。
ただし、断罪・誹謗・煽りを目的とした表現や設計は行わない（「予測↔事実」の照合履歴を追えることを優先する）。

1.3.1.1 観測する予測者（対象範囲の定義・追加・確定）

本プロダクトは「予測を、ファクトで評価する」ため、**観測する予測者（誰の予測を収集するか）** を以下で固定する。

- 原則：一次情報に辿れる形で「誰が・いつ・何を」言ったかを残せるものを観測対象とする
- 例外：一次情報が特定できない場合は、推測で埋めず **"不明"** と明示し、後日一次情報で更新する

観測対象の最小セット（格闘技コンテンツの運用ルール）：

- **Tier A（常に観測）**：当事者性/専門性が高い人物
  - 現役/元トップ選手、主要ジム指導者、著名解説者など
- **Tier B（常に観測）**：格闘技を扱う有名 YouTuber / 配信者 / 解説系発信者
  - 原則基準：登録者数 10 万以上（収集時点）
  - 補完基準：登録者数が基準未満でも、**影響力が高い**と判断できる場合は観測対象に含める
    - 例：業界内での参照頻度が高い、主要選手/主要媒体に継続的に引用される、再生規模や反応規模が継続して大きい、格闘技予測の継続発信実績がある
  - 登録者数・影響力の根拠が曖昧な場合は断定せず、`source_note` に「根拠不明/要確認」と根拠メモを残す

適用範囲（固定）：

- 本条項（観測対象/Tier定義/影響力基準）は **ボクシングに限定せず、格闘技ログ全般**（例：RIZIN / UFC / ボクシング / キックボクシング / MMA など）に適用する
- 種目ごとの差異がある場合も、まず本条項を共通基準として適用し、必要な追加ルールのみを種目別に追記する

補足（データの持ち方）：

- `predictor_name` は **個人名で統一**する
  - 理由：1本の動画に複数人が出演し、それぞれが予測するケースがあるため
- 1本の動画に複数人の予測が含まれる場合：**人物ごとに予測レコードを分ける**
  - `source_url` は同一でもよい（タイムスタンプや `source_note` で該当箇所を分ける）
  - チャンネル名/媒体名は必要に応じて `source_note` に残す（`predictor_name` は個人名を優先）

1.3.1.2 Gemini等への依頼文における予測者スコープ明記（追加・確定）

Gemini等の外部AIへ「予測抽出」または「予測＋結果記載」を依頼する際は、依頼文（プロンプト）に **観測する予測者（誰の予測を取るか）** を毎回明記する。省略しない。

必須で明記する内容（最小セット）：

- Tier A（常に観測）：当事者性/専門性が高い人物（現役/元トップ選手、主要ジム指導者、著名解説者など）
- Tier B（常に観測）：格闘技を扱う有名YouTuber/配信者/解説系発信者（登録者10万人以上を原則。未満でも影響力が高ければ対象）
- `predictor_name` は個人名で統一（媒体名・編集部名・集合ラベルを使わない）
- 1本の動画に複数人が出演する場合は人物ごとに分割
- 個人名を特定できない場合のみ `"不明"` を許容し、`source_note` に要確認理由を残す
- 日本国内の予測・一次情報を優先して収集する（日本語記事/動画/SNS、日本国内メディア、日本で活動する発信者を優先）
- 同条件で比較できる場合は海外ソースより国内ソースを優先し、国内だけで件数要件を満たせない場合のみ海外ソースで補完する

運用上、結果記載を主目的とする依頼であっても、予測レコードを扱う可能性がある場合は本スコープを同様に明記する（「誰の予測を取るか」を曖昧にしない）。

Gemini依頼文の参照先は `docs/MARTIAL_ARTS_GEMINI_PROMPTS.md` を正本とする（旧名称 `docs/BOXING_WORLD_TITLE_GEMINI_PROMPTS.md` は互換扱い）。
件数要件の基準は **最低15件** とし、一次情報が不足する場合のみ不足理由を明記したうえで例外運用を許容する。

1.3.2 予測ソース（出典）の必須化（追加・確定）

信頼性担保のため、予測には必ずソース（一次情報への参照）を記載する。最低限、次のいずれかを満たす：
- URL（動画/記事/投稿など）＋必要ならタイムスタンプや該当箇所のメモ
- リポジトリ内の参照（例：`statements[].statement_id` など、同一ファイル内で一次情報にたどれる識別子）
ソース不明の場合は、推測で埋めず **「不明」** と明示したうえで、後から一次情報に基づいて更新できる形にする。

【運用補足：出典の検証可能性とリンク枯れ（追加・確定）】

本プロダクトが担保するのは **「絶対の真実」ではなく、出典に基づく検証可能性** である（北極星「予測を、ファクトで評価する」と整合する）。読者が一次情報（または当時の写し）に辿って内容を照合できる状態を正とし、**出典なき断定**をしない。

- **URL は枯れる前提** とする。枯れに備え、次を組み合わせて **再現性を上げる**：
  - **ウェブアーカイブ**（例：Internet Archive / Wayback Machine）の URL を、取得できた場合は `source_note` 等に **補助参照**として併記してよい（**正規の一次情報 URL を正本**とし、アーカイブは消滅・改変リスクが別経路にあることを前提とする）。YouTube 等の動画は、アーカイブが **再生・内容の完全な再現**を保証しないことが多い点に留意する。
  - **メタデータ**：人手またはプロセスが出典を確認した日時（**確認日**）を `source_note` 等に残す。推測で補完しない。
- **一次情報に辿れなくなった**（リンク切れかつ利用可能なアーカイブも無い、動画削除など）場合は、推測で体裁だけを整えない。**表現・集計・信頼の置き方を下げる／全文更新で是正する**。具体例：
  - `source_note` に「参照不能・要更新」と理由・確認日を明記する
  - 項目が特定できない場合は **「不明」** と明示する（1.3.1.1・1.3.2 の例外運用に従う）
  - **予測者ハブ等の集計**では、検証可能性を満たさないレコードを **メインの的中率・ランキングの母集団に含めない** 選択を許容する（表示上の扱いは 9.4.2 等の運用メモに従う）。履歴としてレコードを残すかどうかは別途判断し、**残す場合も「当時の記録」として誤解が生じない表現**とする。
- **外部参照（比喩）**：ウィキペディアは **検証可能性** と **出典** を核にし、リンク枯れには **アーカイブ付き引用・編集コミュニティ・ボット** 等で対処する。本プロダクトは編集体の規模は異なるが、**課題の種類（枯れ・再現・更新）は同型** とみなし、**憲法・運用・（将来の）自動チェック** で同種の問題に対処する。

【運用補足：YouTube一次情報の取得について（追加・確定）】

- YouTube 動画からの引用文・タイムスタンプ抽出が必要な場合、補助的に Gemini 等の外部AIを利用してもよい（一次情報URLは必ず動画の正規URLを用いる）
- 外部AIの補助を用いた場合は、`source_note` に「Gemini補助で抽出（確認日：YYYY-MM-DD）」などのメモを残す
- 発言者の特定（個人名）や勝者方向が動画だけで一意に確定できない場合は、推測で埋めずに `speaker.name = "不明"` / `prediction.expected_winner = "不明"` とし、`source_note` に「要確認」と根拠・未確定理由を記載する
- 後日、人手または一次情報の再確認で確定した時点で更新する（全文更新の原則を維持）

【運用補足：Gemini出力の最低検査（追加・確定）】

目的：外部AIの出力が「架空出典」や「別興行のカード混入」を含むことがあるため、一次情報に基づく事実保持を壊さないための最小チェックを固定する。

- Gemini等の出力は **そのまま取り込まず**、次を必ず検査する
  - **URLが実在する一次情報**であること（ダミー/サンプルURLを禁止）
  - URLが **Markdownリンク形式（`[text](url)`）でない**こと（生URLに統一）
  - `prediction.target` が **当該イベントの公式カード表記と一致**すること（別興行/別カード混入を防ぐ）
- 上記に違反するレコードは **破棄または一次情報で再確認して再抽出**する（推測で修正しない）

【運用補足：Gemini に予測抽出を任せる標準フォーマット（追加・確定）】

目的：YouTube/記事/SNS などの一次情報から「勝者方向の予測（誰が勝つ/有利か）」を、`data/events/*.json` の `statements[]`（THE MATCH 型）へ追記できる形で抽出する。

- Gemini の出力は **一次情報URLに遡れる内容のみ**（URL必須、可能なら timestamp）

- `predictor_name` は **個人名で統一**（動画に複数人出演なら人物ごとに分割）
- 予測が無い（勝者方向に言及しない）ものは `statements[]` に入れない（原則）。※例外的に入れる場合は、理由を `source_note` に明記する

1.3.3 予測メタ情報の最小キー（追加・確定）

MVP段階の表示・集計が参照する「予測のメタ情報」は、以下のキーを最小仕様として固定する（このキー名でデータを持つ）。

- 予測者（必須）
  - `predictor_name`: string（例：人名/媒体名/アカウント名）。未確定・未定義の場合は **"不明"** と明示する
- ソース（必須）
  - `source_url`: string（一次情報URL。ある場合はこれを優先）
  - `source_ref`: string（リポジトリ内参照。例：`statements[].statement_id=...`）
  - `source_note`: string（タイムスタンプ/該当箇所メモなど）
  - いずれも不明の場合は、推測で埋めず **"不明"** と明示する

補足：現状 `data/events/*.json` にはスキーマ混在があるため、ランキング（的中率）などの集計対象は「勝者方向が構造化されているデータ」に限定する。混在スキーマを無理に推測で統合しない。
1.4 集計のルール

予測者別集計：予測データに予測者（`predictor_name`）が明記されたものを対象とする（未確定・未定義は **"不明"** と明示して扱う）。
集計単位：興行クローズ時のみ（途中集計で数字を動かさない）。

1.4.1 ランキングの方針（追加・確定）

ランキングは **的中率** に基づく。集計・表示は「興行クローズ時のみ」の原則を維持し、途中経過で数値を動かさない。表示は事実指標としての可視化にとどめ、断罪・誹謗・煽りを目的としない。

1.4.2 的中率の定義（追加・確定）

的中率は **主予測（勝者方向）** の照合結果のみで計算する（副次＝KO/判定/SUB の一致は、必要に応じて別枠の参考情報として扱う）。
集計対象は、`prediction.exists = true` かつ結果（勝者）が確定しているものに限る。
同一興行データに複数種目（例：MMA と KICKBOXING）が混在する場合、**種目別の予測者ハブ上のランキング**では、当該ページの種目に対応する **試合（bout）単位の `discipline`** に合致する予測のみを集計する（発言ログは `prediction.target` とカード表記の照合で bout に紐づけて同様に判定する）。種目をまたいだ混在集計をしない。
1.5 会場スコープの優先順位

判定は 会場 → スコープ の順で決める。
一度決めたスコープ判定は 逆転不可。
1.6 「ドームクラス興行」の定義

ドームクラス = 東京ドーム開催のみ（独自定義）。
1.7 MVP段階での制約

MVP段階では indexes 作成禁止。
