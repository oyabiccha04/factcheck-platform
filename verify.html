検証用データ・リポジトリ

生のJSON・GitHub は、一般の閲覧より機械処理・一次データの確認・変更履歴の追跡向けです。通常はトップの「イベント詳細」や各ログ・予測ページからお読みください。JSON 内の外部 URL はリンク枯れしうるため、検証時は確認日・アーカイブ併記の有無もあわせて参照してください（方針は PROJECT_CHARTER.md）。

データの再利用・クレジット・免責については ポリシー・利用条件 を参照してください。

機械可読インデックス

  • data/index.json（全イベント一覧のマニフェスト）

GitHub Pages ではキャッシュが残ることがあるため、更新直後は URL に ?v=日付 などを付与すると確実です。

興行ごとのイベント JSON

各ファイルは data/events/<ファイル名>.json にあります。インデックスの file フィールドと対応します。

  • 例: the-match-2022.json（実在ファイルに置き換えて利用）

画面上で試合カード・発言ログを読む場合は event.html（?file=…）を利用してください。

ソースコード・変更履歴

  • GitHub リポジトリ（commit 履歴・Issue 等）

プロダクトの前提・運用ルールはリポジトリの PROJECT_CHARTER.md を参照してください。
