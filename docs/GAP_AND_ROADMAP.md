# 齟齬整理と格闘技コンテンツ強化のロードマップ

## 1. 現状で起きている齟齬

### 1.1 インデックスと実ファイルの不一致
| 状況 | 詳細 |
|------|------|
| **data/index.json** | RIZIN 20〜35 の **16件のみ** 登録 |
| **data/events/** | 上記に加え **rizin-40.json** / **the-match-2022.json** が存在 |
| **結果** | 一覧（index / logs）からは 40 と THE MATCH 2022 に到達できない |

→ **対応**: `data/index.json` に `rizin-40` と `the-match-2022` を追加する。

---

### 1.2 イベントJSONのスキーマが2系統ある
| 系統 | ファイル例 | 中身 | 現在の event.html |
|------|------------|------|-------------------|
| **A: 試合結果型** | rizin-20〜35 | `event_meta` + `cards[].bouts[]`（red/blue/outcome） | ✅ 試合一覧として表示できる |
| **B: 発言・予測型** | the-match-2022 | `event_name` / `event_date` / **`statements[]`**（prediction + verification） | ❌ 未対応。「試合データがありません」になる |

**the-match-2022.json** には「誰が何と予測し、実際はどうなったか」が入っているが、**event.html は B 型を読んでおらず、予測→検証を一切表示していない**。

→ **対応**: event.html で「statements 型」を扱い、**発言・予測** と **検証結果** を同じページに表示する。

---

### 1.3 北極星と画面のギャップ
| CHARTER の価値 | 現状の画面 |
|----------------|------------|
| 「予測を、ファクトで評価する。」 | トップのヒーローコピーになっていない（タイトルは "factcheck-platform" のまま） |
| 予測と結果の**照合**を残す | event ページは「試合結果一覧」のみ。**予測とその検証**を並べて見せる箇所がない |
| 格闘技は起点コンテンツ | トップから「格闘技ログ」への導線がない（martial-arts/ は存在するがリンクなし） |

→ **対応**: トップに北極星コピーと格闘技入口を出す。event で「予測→結果」が 1 件でも見えるようにする。

---

### 1.4 導線の二重構造
- **メイン**: index.html → logs.html → event.html（全団体・data/index.json ベース）
- **格闘技専用**: martial-arts/index.html → martial-arts/rizin.html（RIZIN のみ・同じ data/index.json を filter）

index から martial-arts へのリンクがなく、**格闘技を入口にしたい人**がたどり着きにくい。

→ **対応**: index.html の「入口」に「格闘技ログ（RIZIN など）」へのリンクを追加する。

---

## 2. 格闘技を「面白い」と感じるために必要なこと

サイトの差別化は **「誰が何と予測し、結果はどうだったか」が一目でわかる** こと。

1. **予測と結果を同じ画面で見せる**  
   - 少なくとも **the-match-2022** を開いたときに、発言・予測と検証結果が表示されるようにする。
2. **トップで「予測を、ファクトで評価する」を打ち出す**  
   - タイトルやリードに北極星を入れ、格闘技ログへの導線を 1 クリックにする。
3. **データの穴を埋める**  
   - index.json に未登録のイベント（rizin-40, the-match-2022）を追加し、どこからでもたどり着けるようにする。

---

## 3. 推奨アクション（優先順）

| 優先度 | やること | 効果 |
|--------|----------|------|
| 1 | **data/index.json** に `rizin-40` と `the-match-2022` を追加 | 一覧と実データの齟齬解消 |
| 2 | **event.html** で statements 型（prediction / verification）を表示 | 「予測→結果」が 1 件でも見え、北極星が体感できる |
| 3 | **index.html** に北極星コピーと「格闘技ログ」リンクを追加 | 初見で何のサイトか分かり、格闘技にすぐ行ける |
| 4 | （任意）RIZIN の bout に「事前予測」を載せるスキーマ拡張や、the-match 型イベントを増やす | コンテンツ量が増え、面白さが持続する |

---

## 4. 触るファイルの目安

- **データ**: `data/index.json`（2件追加）
- **表示**: `event.html`（statements 型の描画ブロック追加）
- **入口**: `index.html`（タイトル・リード・格闘技リンク）

この順で進めると、齟齬解消と「格闘技の予測を面白く見せる」の両方が達成しやすい。
